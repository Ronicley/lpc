Tags de Texto:

<b> Texto em negrito;
<strog> Negrito em HTML5;
<i> Maior ênfase em itálico, <em> Texto em itálico;
<u> Texto sublinhado
<sup> Texto subscrito, <sub>, texto sobrescrito;
<br/> Representa querbra de linha.
<hx> São elementos que representam os seis níveis de titulos de
cabeçalhos dos documentos. Um elemento título descreve brevemente o tema da seção.
<blockquote> Representa uma citação.
<pre> Indica que o seu conteúdo é pré-formatado e que este formato deve ser preservada.
<hr> Representa uma ruptura temática entre parágrafos de um artigo ou seção ou qualquer conteúdo mais longo.

Tags de Listas:

<ol>

Representa uma lista ordenada de itens. Tipicamente, listas ordenandas
de items são mostradas com um número precente, o qual pode ser de qualquer tipo,
como números, letras, numeros romanos ou ainda simples símbolos. Esse estilo
numerado não é definido no html da página, mas sim no CSS associado, usando a propriedade list-style-type.

<ul>

Representa uma lista desordenada de itens, ou seja, uma coleção de itens que não têm uma ordem numérica,
e sua ordem na lista é sem sentido. Normalmente, os itens de lista desordenada são
exibidas com um simbolo, que pode ser de várias formas, como um ponto, um círculo
ou um quadrado. O estilo de marcador não é definido na descrição do HTML da página,
mas em seu CSS associada, utilizando a propriedade list-style-type

<li>

O HTML List item element (<li>) é usado para representar um item de
lista. Ele deve estar contido em uma lista ordenada (<ol>), uma lista não ordenada
(<ul>) ou um menu (<menu>), onde representa uma única entidade na referida lista.
Em listas ordenadas , eles são geralmente
exibido com alguns contador ascendente do lado esquerdo , como um número ou letra.

Tags de Links:

<link>

Especifica as relações entre o documento atual e um recurso externo.
Possíveis usos para este elemento incluem a definição de uma estrutura
relacional para navegação. Este elemento é mais usado para vincular as folhas de estilo.

Tags de Imagens:

<img>

representa a inseração de imagem no documento, sedo implementado
também pelo HTML5 para uma melhor experiência
com o elemento <figure> e <figcaption>.

Tags de Tabelas:

<div>

Representa um contêiner genérico sem nenhum significado especial.

<span>

Representa Representa o texto com nenhum significado específico.
Isso tem que ser usado quando nenhum outro elemento do texto-semântico transmite
um significado adequado, o que, neste caso, é muitas vezes trazido por atributos globais,
como classe, lang, ou dir.

Diferenças entre < div > e < span > :

Ambos são tags utilizadas para agrupar blocos HTML e aplicar alguma função a esse bloco.
A diferença entre os dois é que span é utilizado para um pequeno bloco de HTML, e é um elemento inline (Ocupa apenas o espaço necessário e não inicia uma nova linha no documento. São chamados elementos em linha justamente por aparecer na mesma linha que outros elementos, caso seja possível).
Já o div é usado para agrupar grupos maiores de código e é um elemento block o que basicamente é equivalente a ter uma quebra de linha antes e depois do elemento.

NOVOS ELEMENTOS

<article>

Conteúdo autossuficiente;
Exemplos:
Post em um fórum;
Artigo de revista ou jornal;
Matéria de um blog;
Comentário de um visitante;
Pode ser aninhados, desde que exista realmente a dependência
semântica;
Exemplo uma resposta em um comentário;

<section>

Marca uma seção na página;
Seção é um grupamento de conteúdo tratando de um
mesmo assunto;
Geralmente possui um header e um footer

<nav>

Demarcar uma seção da página que contenha grupamentos
de links para navegação;

<mark>

Destacar um trecho importante para o contexto atual;
Efeito “marca texto”;

<Header>

Mesmo funcionamento do footer, só que para cabeçalhos;

<Footer>

Elemento para marcar o rodapé de uma seção ou página;

<Figure> <FigCaption>

Fornece um container para elementos que, embora sejam
relacionados com o conteúdo principal, podem ser movidos
ou reposicionados sem prejuízo do entendimento;
Exemplo:
Imagens;
Vídeos;
Citações;
Diagramas;

<Aside>

Conteúdo relacionado ao assunto principal;
Embora relacionado, sua retirada não prejudica a compreensão
do conteúdo principal;
Exemplo:
Barras laterais;
Banners;
Textos com propaganda;
Agrupamento de links relacionados;

FORMULARIO

<form>

  • Usada para coletar dados do usuário
    1. <input>:Representa um campo do formulário;
      1. Atributo “name” obrigatório;
        Pode ser dos tipos (atributo “type”):
        1. text;
        2. password;
        3. submit;
        4. radio;
        5. checkbox;
        6. button;
        7. number;
        8. range;
    2. <select>
      1. Cria um drop down list;
      2. Opções com a tag option;
    3. <textarea>
      1. Campo com várias linhas
    4. <button>
      1. Cria um botão;
      2. Não faz o submit automático do formulário;
      3. Pode conter imagem no interior do botão;
